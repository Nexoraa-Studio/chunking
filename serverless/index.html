Architecture

Browser → API Gateway HTTP API → Lambda chunking-api. Upload bypasses Lambda (browser POSTs directly to S3 with a presigned URL). On Run, Lambda calls ecs:RunTask on task-def chunking-worker:1 (1 vCPU / 4 GB). Worker pulls PDF from S3, runs pymupdf → structural → coarse-DP, uploads every artifact (elements/structural/coarse JSONL, metrics JSONs, chunks.zip, status.json) back to S3. Download is a 302 to a presigned GET URL.

Cost profile (idle)

Lambda + APIGW + S3 ≈ $1/mo idle. Fargate billed only while a task is running (≈ $0.058/hr). Compare: ECS Express always-on ALB is $16/mo even paused.

What's NOT in this UI (yet)

Live log: we DO show CloudWatch Logs events polled every 3 s. Not technically SSE; API Gateway HTTP API doesn't support SSE cleanly. The 3-second polling cadence is within a second or two of true "live" for the task's own stdout.

Evaluation / UMAP figures: skipped because we'd need ground-truth queries per PDF, and the viz step (matplotlib + umap-learn) would bloat the Fargate image. The ECS Express dashboard has them since it runs locally against a stable artifact set.
